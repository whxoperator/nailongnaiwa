## 4. 运行方式

### 4.1 安装依赖

```powershell
pip install -r requirements.txt
```

### 4.2 训练模型

```powershell
python train_nailong_naiwa.py --cpu --epochs 12 --repeats 3 --batch-size 16 --image-size 128 --print-every 3 --out models/nailong_naiwa_balanced_cnn.pt
```

训练脚本会构建平衡实验数据集，生成训练集、测试集和泛化集划分，并保存训练后的模型文件。

### 4.3 微调预训练模型

```powershell
python train_pretrained_classifier.py --cpu --arch resnet18 --epochs 10 --batch-size 16 --image-size 160 --freeze-backbone --out models/nailong_naiwa_resnet18_head.pt
```

该脚本加载 ImageNet 预训练的 ResNet18 骨干网络，将最后的分类头替换为奶龙 / 奶蛙二分类任务头，并在本项目小规模数据集上进行微调。通过 `--random-init` 可以与同结构随机初始化模型进行对照。

### 4.4 启动交互界面

```powershell
python predict_nailong_naiwa_ui.py --host 127.0.0.1 --port 8770 --cpu
```

浏览器打开：

```text
http://127.0.0.1:8770
```

界面支持上传图片、选择 CNN 模型、选择传统图像算法、查看预测类别和置信度，也支持对同一张图片进行多算法对比。

如需启用云端多模态 LLM 选项，可在启动前配置 OpenAI-compatible API：

```powershell
$env:OPENAI_API_KEY="your_api_key"
$env:OPENAI_MODEL="gpt-4o-mini"
python predict_nailong_naiwa_ui.py --host 127.0.0.1 --port 8770 --cpu
```

如果使用其他兼容 OpenAI 接口的服务，可额外设置 `OPENAI_BASE_URL`。未配置 API Key 时，CNN 和传统算法仍可正常使用，LLM 选项会提示未启用。

### 4.5 批量算法对比

```powershell
python compare_nailong_algorithms.py --data nailong_naiwa_splits/test --split-dir nailong_naiwa_splits --models-dir models --out-dir reports --cpu
```

运行后会生成：

- `reports/algorithm_comparison.md`
- `reports/algorithm_comparison.json`
- `reports/algorithm_predictions.csv`

## 5. 实验结果展示

当前测试集算法对比结果如下：

| 算法 | 准确率 |
| --- | ---: |
| CNN balanced model | 90.24% |
| Thumbnail kNN | 87.80% |
| RGB histogram prototype | 85.37% |
| Color mean prototype | 78.05% |
| Edge histogram prototype | 58.54% |
| ResNet18 pretrained head | 48.78% |

实验结果表明，在当前小规模数据集上，平衡数据训练得到的 CNN 模型效果最好；传统图像算法也可以作为对照基线，用于判断深度学习模型是否真正学习到有效特征。预训练 ResNet18 只冻结骨干并训练分类头时效果有限，说明小数据集上的迁移学习仍需要继续调整学习率、解冻层数和训练轮数。

## 6. 数据来源说明

本项目数据主要来自两个部分：

1. GitHub 开源项目 `spawner1145/NailongRecognize`。
2. 从小红书等平台下载并整理得到的奶龙 / 奶蛙图片。

由于图片来源不同，项目中将数据划分为：

- `train`：训练集，用于模型学习；
- `test`：测试集，用于固定评估；
- `generalization`：泛化集，用于观察模型面对不同来源图片时的表现。

## 8. 项目不足与后续计划

当前项目仍有以下不足：

- 数据集规模较小，测试集数量有限，准确率存在一定偶然性。
- 分类类别目前只有奶龙和奶蛙，无法识别其他角色或无关图片。
- 模型可能受到背景、水印、压缩质量和图片来源影响。
- Web UI 仍偏实验演示，尚未实现完整的批量上传、历史记录、用户管理和人工审核功能。
- 原计划中的云端数据库、多人协作数据管理和预测历史保存功能因时间原因未能完全实现。

后续可以继续扩展：

- 扩充数据集；
- 增加错误样本分析；
- 优化 Web UI；
- 建立云端数据库；
- 引入更多评估指标；
- 继续调优迁移学习策略或更强的预训练模型；
- 扩展云端多模态 LLM 的结构化解释、预测历史保存和人工复核流程。
